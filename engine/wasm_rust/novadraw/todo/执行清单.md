# Novadraw 执行清单

## 项目目标

实现类似 Eclipse Draw2D/GEF 的矢量图绘图引擎，参考 draw2d 架构设计，具有良好的扩展性。

---

## 当前架构

```
┌─────────────────────────────────────────────────────────────┐
│                      novadraw (workspace)                    │
├─────────────────────────────────────────────────────────────┤
│ Level 1: 核心库（无运行时依赖）                              │
│   ├── novadraw-math/  - Vec2, Vec3, Mat3, Rect, Transform  │
│   └── novadraw-core/  - Color, Id                          │
├─────────────────────────────────────────────────────────────┤
│ Level 2: 渲染层                                             │
│   └── novadraw-render/ - RenderCommand, NdCanvas, Vello    │
├─────────────────────────────────────────────────────────────┤
│ Level 3: 场景图（Draw2D 定位）                              │
│   └── novadraw-scene/  - Figure, SceneGraph, Viewport      │
├─────────────────────────────────────────────────────────────┤
│ Level 4: 编辑器（GEF 定位）                                 │
│   └── apps/editor/    - SceneManager, 事件处理              │
└─────────────────────────────────────────────────────────────┘
```

### Phase 9 详解

#### 9.1 R-Tree 空间索引

**背景**：当前 `hit_test` 使用全量遍历 O(n)，大量图形时性能差

**实现方案**：

- 在 `SceneGraph` 中添加 `spatial_index` 字段
- 存储 Figure 的 bounds（世界坐标）
- 查询时：屏幕坐标 → 世界坐标 → R-Tree 查询 → contains_point 精确检测

**增量添加，不影响现有 API**：

```rust
// 添加到 SceneGraph
pub struct SceneGraph {
    // ... 现有字段
    spatial_index: Option<RTree<SpatialEntry>>,  // 新增
}

// 修改 hit_test 实现
pub fn hit_test(&self, point: (f64, f64)) -> Option<BlockId> {
    // 1. R-Tree 快速初筛（返回候选）
    // 2. contains_point 精确确认
}
```

**更新时机**：

| 操作 | R-Tree 变化 |
|------|-------------|
| add_child_to | 插入 |
| prim_translate | 更新 |
| set_bounds | 更新 |
| delete | 删除 |
| viewport 变化 | **不需要** |

**推荐 crate**：`rstar`（Pure Rust）
