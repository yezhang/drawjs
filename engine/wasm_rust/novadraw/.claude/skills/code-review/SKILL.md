# 代码审查指南

## 审查要点

### 1. 正确性检查

- [ ] 代码逻辑是否正确
- [ ] 边界条件是否处理
- [ ] 错误处理是否完善
- [ ] 是否有潜在的 panic 风险

### 2. Rust 最佳实践

- [ ] 是否使用 `?` 代替 `unwrap()`
- [ ] 是否有不必要的 `unsafe`
- [ ] 生命周期标注是否正确
- [ ] 是否遵循 Rust 命名规范

### 3. 性能考虑

- [ ] 是否有不必要的内存分配
- [ ] 循环中是否有重复计算
- [ ] 是否使用了高效的数据结构

### 4. 架构设计

- [ ] 是否遵循项目模块划分
- [ ] trait 定义是否合理
- [ ] 是否符合核心原则（Figure 不包含状态）

### 5. 文档与测试

- [ ] 公开 API 是否有文档注释
- [ ] 复杂逻辑是否有注释
- [ ] 核心算法是否有单元测试

### 6. 安全性

- [ ] 是否有潜在的溢出风险
- [ ] 是否有未初始化的内存
- [ ] 是否正确处理了并发访问

## 审查输出格式

```markdown
## 代码审查报告

### 文件
`src/path/to/file.rs`

### 问题

#### 高优先级
1. **[问题描述]** - 位置
   - 原因：...
   - 建议：...

#### 中优先级
...

#### 低优先级
...

### 优点
- ...

### 总体评价
[优秀/良好/需要改进]
```

## 常见问题模式

| 问题模式 | 检查项 | 建议 |
|----------|--------|------|
| `unwrap()` | 关键路径 | 使用 `?` 或 `match` |
| `unsafe` | 必要性 | 确认无法用安全代码实现 |
| 递归 | 树遍历 | 改用迭代实现 |
| `println!` | 热路径 | 移除或使用日志框架 |
| magic numbers | 常量 | 定义为常量并命名 |
| 重复代码 | 抽象 | 提取为函数或宏 |

## 审查命令

```bash
# 运行 clippy
cargo clippy -- -D warnings

# 运行测试
cargo test

# 检查文档
cargo doc --no-deps

# 运行 miri 检查未定义行为
cargo +nightly miri test
```
